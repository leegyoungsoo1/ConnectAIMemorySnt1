### 📋 오탐 발생 핵심 원인 및 평가

1. **기술 스택 식별 실패 (Stack Misidentification)** ⚠️
   - 대상 사이트는 **ASP.NET / IIS** 환경임에도 불구하고, 스캔 도구 또는 분석 프로세스가 **PHP / WordPress / Node.js** 기반의 범용 체크리스트를 그대로 적용하였습니다. [근거: phpinfo.php 및 /wp-admin 탐지 결과]
   
2. **정적 경로 기반의 단순 매칭 (Static Path Matching)** ⚠️
   - 실제 파일의 존재 여부와 응답 값의 내용을 정밀 분석하지 않고, 특정 경로(/wp-admin, .env 등)에 대한 요청 후 단순 응답 코드를 기반으로 취약점을 판정하는 오류가 발생하였습니다. [근거: .git, .env 오탐]

3. **환경 특성 반영 미흡 (Environment Specifics)** ⚠️
   - IIS의 `web.config` 접근 차단 기본 설정 및 이미 적용된 AES-256 암호화 설정을 분석 과정에서 누락하였습니다. [근거: web.config 판정]

### ✅ 향후 조치 및 개선 방향

- **맞춤형 체크리스트 적용:** 범용 스캔 대신 **.NET Framework / IIS 전용 보안 가이드라인**으로 체크리스트를 즉시 교체하겠습니다.
- **검증 단계 강화:** 취약점 발견 시 단순히 HTTP 상태 코드만 확인하지 않고, 응답 본문의 실제 내용(Body)을 대조하는 검증 단계를 추가하여 오탐률을 낮추겠습니다.
- **자산 식별 우선 수행:** 분석 시작 전 `Fingerprinting` 단계를 통해 정확한 OS, 웹 서버, 언어 버전을 먼저 확정한 후 스캔 시나리오를 설계하겠습니다.
