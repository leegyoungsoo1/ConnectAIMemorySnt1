# 📋 작업 브리프

**원 명령:** 다음부터는 웹사이트 점검전에 이 웹사이트가 어떤 것으로 만들어졌는지 부터 먼저 파악을 하고 분석해야 될것 같아. 지금 사이트는 .NET Framework , iis 이지만 다른 사이트는 또 다를수 있으니 유동적으로 잘 판단해서 처리하도록 해.

## 요약
향후 모든 웹사이트 점검 시, 분석 전 대상 사이트의 기술 스택(OS, 서버, 언어 등)을 우선 식별하는 'Fingerprinting' 프로세스를 표준 절차로 수립합니다.

## 분배
- **🌐 경수**: 웹사이트 점검 프로세스 최상단에 '기술 스택 식별(Fingerprinting)' 단계를 추가하십시오. 분석 대상의 서버(IIS, Nginx, Apache 등)와 프레임워크(.NET, PHP, Java 등)를 먼저 확정하고, 그에 맞는 전용 보안 체크리스트를 선택하여 분석하는 가이드를 수립하여 팀장님께 보고하십시오.
- **📋 영철**: 범용 체크리스트가 아닌 기술 스택별(예: .NET/IIS 전용, PHP/Linux 전용) 맞춤형 취약점 분류 체계를 정리하고, 분석 결과 보고서에 '식별된 기술 스택' 항목을 필수 기재하도록 보고서 양식을 업데이트하여 팀장님께 보고하십시오.
