# 🛒 고객사 쇼핑몰 웹사이트 개발 프로젝트 계획서

## 1. 프로젝트 개요
- **프로젝트 명:** 고객사 맞춤형 반응형 쇼핑몰 구축
- **목표:** 현대적인 UI/UX가 적용된 반응형 쇼핑몰 구현 및 강력한 보안 체계 구축
- **핵심 가치:** 사용자 편의성(UX), 크로스 브라우저 호환성, 보안 무결성, 확장성

## 2. 기술 스택 (Proposed)
### Frontend
- **Framework:** React.js (TypeScript)
- **Styling:** Tailwind CSS (반응형 UI 최적화)
- **State Management:** Zustand 또는 Redux Toolkit
- **Build Tool:** Vite

### Backend
- **Language/Framework:** Node.js (NestJS) 또는 Python (FastAPI)
- **Database:** PostgreSQL (상품/주문), Redis (캐싱/세션)
- **Infrastructure:** AWS (EC2, S3, RDS)

### Security
- **Authentication:** JWT (JSON Web Token) & OAuth 2.0
- **Validation:** 서버 사이드 이중 검증 체계
- **Testing:** OWASP Top 10 기반 취약점 점검

## 3. 상세 개발 일정 (8주 공정)

| 주차 | 단계 | 프론트엔드 (지현) | 백엔드 (개발팀) | 보안 점검 (경수/영호) | 비고 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **1주** | **분석 및 설계** | UI/UX 와이어프레임, 디자인 시스템 정의 | DB 스키마 설계, API 명세서 작성 | 보안 요구사항 정의 및 위협 모델링 | |
| **2주** | **기초 구현** | 프로젝트 환경 설정, 공통 컴포넌트 개발 | 인증/인가(Auth) 서버 구현, API 기본 구조 | 인증 로직 취약점 분석 (Auth Bypass 점검) | |
| **3주** | **핵심 기능 A** | 메인 페이지, 상품 리스트, 필터링 UI | 상품 관리 API, 카테고리 검색 기능 | Input Validation 및 SQL Injection 점검 | |
| **4주** | **핵심 기능 B** | 상품 상세 페이지, 장바구니, 찜 기능 | 장바구니 로직, 주문/결제 API 기초 | 권한 관리(IDOR) 취약점 점검 | |
| **5주** | **핵심 기능 C** | 주문/결제 프로세스 UI, 마이페이지 | 결제 게이트웨이 연동, 주문 처리 시스템 | 결제 로직 무결성 및 파라미터 변조 점검 | |
| **6주** | **고도화 및 최적화** | 반응형 최적화, 애니메이션, 웹 접근성 준수 | 성능 최적화, API 캐싱 적용 | XSS 및 CSRF 방어 체계 전수 점검 | |
| **7주** | **통합 테스트** | 프론트-백엔드 API 연동 및 QA | 통합 API 테스트 및 버그 수정 | **전체 모의해킹 및 침투 테스트 수행** | |
| **8주** | **안정화 및 납품** | 최종 UI 폴리싱, 브라우저 호환성 테스트 | 서버 배포 및 환경 설정 최적화 | 최종 보안 이행 점검 및 보고서 제출 | |

## 4. 보안 점검 상세 전략
단순히 마지막에 점검하는 것이 아니라, **'개발 $\rightarrow$ 점검 $\rightarrow$ 수정'**의 사이클을 반복합니다.

1. **설계 단계:** 위협 모델링을 통해 잠재적 취약점 사전 식별
2. **구현 단계:** 
   - 프론트엔드: `textContent` 사용, 입력값 필터링, CSP 설정
   - 백엔드: 파라미터화 쿼리 적용, 서버 사이드 검증 강제
3. **검증 단계:** 
   - OWASP Top 10 기준 체크리스트 점검
   - RCE(원격 코드 실행) 가능성 차단을 위한 파일 업로드 경로 및 권한 제어 검증
4. **최종 단계:** 실제 공격 시나리오 기반의 모의해킹 수행 및 조치 결과 확인

## 5. 기대 효과
- **사용자:** 기기 상관없이 쾌적한 쇼핑 경험 제공
- **운영자:** 보안 사고 걱정 없는 안정적인 서비스 운영
- **기업:** 보안성과 완성도를 모두 갖춘 포트폴리오 확보 및 고객 신뢰도 상승
