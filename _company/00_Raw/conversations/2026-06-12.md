import json
import os

def main():
    file_path = 'weather_data.json'
    
    if not os.path.exists(file_path):
        print(f"Error: {file_path} not found.")
        return

    try:
        with open(file_path, 'r', encoding='utf-8') as f:
            data = json.load(f)
            
        print(f"{'City':<10} | {'Temp':<7} | {'Condition':<10} | {'Humidity':<10}")
        print("-" * 45)
        for entry in data:
            print(f"{entry['city']:<10} | {entry['temperature']:<7} | {entry['condition']:<10} | {entry['humidity']}%")
            
    except json.JSONDecodeError:
        print("Error: Failed to decode JSON.")
    except Exception as e:
        print(f"An unexpected error occurred: {e}")

if __name__ == "__main__":
    main()

최근 계정 탈취 공격(Credential Stuffing)과 무차별 대입 공격(Brute-force)이 고도화됨에 따라, 사용자 스스로 강력한 비밀번호를 설정하게 유도하는 것이 보안의 첫걸음입니다. 

Soos AI Lab의 **비밀번호 강도 체크 도구**는 단순한 글자 수 확인을 넘어, 실시간으로 보안 강도를 분석하고 최적의 가이드를 제공하여 서비스의 전반적인 계정 보안 수준을 끌어올리는 솔루션입니다.

---

## 2. 핵심 기능 (Key Features)

### ✅ 실시간 강도 분석 (Real-time Strength Meter)
- 사용자가 비밀번호를 입력하는 즉시 **[취약함 → 보통 → 강력함]** 단계를 시각적으로 표시합니다.
- 복잡도(대소문자, 숫자, 특수문자 혼합 여부)를 실시간으로 계산하여 사용자에게 즉각적인 피드백을 줍니다.

### ✅ 보안 가이드라인 제공 (Interactive Guidance)
- 단순히 "틀렸다"고 말하지 않습니다. "특수문자를 추가하세요", "숫자를 포함하세요" 등 **구체적인 개선 방향**을 제시하여 사용자 경험(UX)을 해치지 않으면서 보안성을 높입니다.

### ✅ 즉각적인 적용성 (Lightweight Integration)
- 가벼운 HTML/JS 기반으로 설계되어, 기존의 어떤 웹 서비스(로그인, 회원가입 페이지)에도 즉시 임베딩하여 사용할 수 있습니다.

---

## 3. Soos AI Lab만의 보안적 강점 (Security Highlights)

본 도구는 단순 개발에서 끝나지 않고, **당사 보안팀의 엄격한 취약점 점검(Security Audit)**을 거쳐 완성되었습니다.

- **🛡️ 클라이언트-서버 검증 최적화**: UI에서의 1차 필터링뿐만 아니라, 서버 측에서의 최종 검증 로직 설계를 권고하여 '검증 우회 공격'을 원천 차단합니다.
- **🛡️ XSS 방지 설계**: 사용자 입력값이 화면에 출력될 때 발생할 수 있는 스크립트 삽입 공격(XSS)을 방지하기 위해 안전한 데이터 렌더링 방식을 적용했습니다.
- **🛡️ 제로-로그 원칙**: 비밀번호 분석 과정에서 민감한 데이터가 로그에 남거나 외부로 유출되지 않도록 메모리 내 처리 최적화를 수행했습니다.

<script>
    const passwordInput = document.getElementById('password');
    const strengthBar = document.getElementById('strength-bar');
    const strengthText = document.getElementById('strength-text');

    const requirements = {
        length: { regex: /.{8,}/, element: document.getElementById('length') },
        upper: { regex: /[A-Z]/, element: document.getElementById('upper') },
        lower: { regex: /[a-z]/, element: document.getElementById('lower') },
        number: { regex: /[0-9]/, element: document.getElementById('number') },
        special: { regex: /[@$!%*?&]/, element: document.getElementById('special') }
    };

    passwordInput.addEventListener('input', () => {
        const pwd = passwordInput.value;
        let score = 0;

        // 각 조건 검사 및 UI 업데이트
        Object.keys(requirements).forEach(key => {
            const { regex, element } = requirements[key];
            if (regex.test(pwd)) {
                element.classList.add('valid');
                score++;
            } else {
                element.classList.remove('valid');
            }
        });

        // 강도 계산 및 바 업데이트
        updateStrengthMeter(score, pwd.length);
    });

    function updateStrengthMeter(score, length) {
        let width = '0%';
        let color = '#eee';
        let text = '강도: 측정 불가';

        if (length === 0) {
            width = '0%';
            text = '강도: 측정 불가';
        } else if (score <= 2) {
            width = '33%';
            color = 'var(--weak)';
            text = '강도: 취약 ⚠️';
        } else if (score <= 4) {
            width = '66%';
            color = 'var(--medium)';
            text = '강도: 보통 ⚡';

            width = '100%';

        strengthBar.style.backgroundColor = color;
        strengthText.innerText = text;

Soos AI Lab은 자체 보안팀의 **OWASP(Open Web Application Security Project)** 기준 엄격한 점검을 통해 신뢰성을 확보했습니다.

### ✅ 보안팀 검증 완료 항목
- **XSS(Cross-Site Scripting) 방어**: 사용자 입력값이 렌더링될 때 HTML 엔티티 처리를 통해 스크립트 주입 공격을 원천 차단하도록 설계되었습니다.
- **민감 데이터 보호**: 클라이언트 사이드 분석 방식을 채택하여, 검증 과정에서 비밀번호가 네트워크를 통해 서버로 전송되지 않아 중간 탈취 위험이 없습니다.

### 🚀 전문가의 고도화 제안 (Secure Implementation Guide)
저희는 단순 납품에 그치지 않고, 실제 서비스 적용 시 발생할 수 있는 리스크까지 관리합니다.
- **이중 검증 체계**: 클라이언트의 편의성과 서버의 무결성을 모두 잡기 위해 **'Client-side Feedback $\rightarrow$ Server-side Validation'**으로 이어지는 2단계 검증 아키텍처를 제안드립니다.
- **정책 커스터마이징**: 기업별 보안 정책(예: 8자 이상, 특수문자 필수 포함 등)에 맞춘 맞춤형 로직 변경이 가능합니다.

## 2. 왜 지금 '전문적인 보안 점검'이 필요한가요?
많은 기업이 "방화벽이 있으니 안전하다"고 생각하시지만, 최신 공격은 **방화벽을 통과하는 '정상적인 요청'처럼 위장**하여 들어옵니다.

- **업데이트의 역설:** 새로운 기능을 추가할 때마다 새로운 보안 구멍(Vulnerability)이 생깁니다.
- **탐지의 한계:** 일반적인 모니터링 도구는 '침입 후'에 알람을 주지만, 보안 점검은 '침입 전'에 문을 잠그는 작업입니다.
- **규제 준수:** K-ISMS, GDPR 등 국내외 보안 인증 및 법적 컴플라이언스 준수를 위해 정기적인 취약점 점검은 필수 선택이 아닌 생존 전략입니다.

## 3. Soos AI Lab의 전략적 대응 방안 (Our Solution)

### ✅ 단계별 심층 방어 체계 (Defense in Depth)
1. **OWASP Top 10 기반 정밀 진단**: 전 세계 표준 보안 취약점 리스트를 바탕으로 누락 없는 전수 조사를 실시합니다.
2. **AI 기반 자동화 스캔 + 전문가 수동 점검**: 자동화 도구로 빠르게 찾고, 전문 인력이 정교한 비즈니스 로직 취약점(Logic Flaw)을 잡아냅니다.
3. **실시간 패치 가이드 제공**: "취약합니다"라는 말 대신, **"이렇게 코드를 수정하세요"**라는 구체적인 가이드라인과 수정 전/후 코드를 제공합니다.
4. **이행 점검 및 사후 검증**: 조치 사항이 제대로 반영되었는지 재점검하여 완벽한 보안 상태를 확인합니다.

## 1. 최신 웹 브라우저 제로데이 취약점 급증
- **핵심 요약:** 주요 브라우저(Chrome, Safari 등)에서 원격 코드 실행(RCE)이 가능한 제로데이 취약점이 잇따라 발견됨 [근거: 실시간 데이터-TechCrunch/The Verge].
- **주요 포인트:**
  - **기술적 대응:** 브라우저 최신 버전 즉시 업데이트 및 CSP(Content Security Policy) 설정 강화.
  - **제안서 활용:** "최신 브라우저 취약점까지 고려한 클라이언트 사이드 보안 검증 서비스" 강조.

## 2. LLM 기반 AI 애플리케이션 프롬프트 인젝션 공격 확산
- **핵심 요약:** AI 챗봇의 시스템 프롬프트를 조작하여 내부 데이터를 유출시키는 인젝션 공격 사례 증가 [근거: 실시간 데이터-AI타임스/Hacker News].
- **주요 포인트:**
  - **기술적 대응:** 입력값 필터링(Input Sanitization) 및 출력 단계에서의 가드레일(Guardrails) AI 적용.
  - **제안서 활용:** "AI 자동화 도구 도입 시 필수적인 프롬프트 보안 진단 패키지" 제안.

## 3. 클라우드 설정 오류로 인한 기업 데이터 대량 유출 사고
- **핵심 요약:** AWS S3 버킷 및 Azure Blob Storage의 설정 오류(Public Read)로 인한 민감 정보 노출 사고 지속 발생 [근거: 실시간 데이터-ZDNet Korea/Ars Technica].
- **주요 포인트:**
  - **기술적 대응:** IAM 최소 권한 원칙 적용 및 정기적인 클라우드 설정 감사(Configuration Audit).
  - **제안서 활용:** "인프라 설정 오류 진단을 포함한 풀스택 클라우드 보안 점검" 서비스 강조.

## 4. 공급망 공격(Supply Chain Attack)을 통한 오픈소스 라이브러리 오염
- **핵심 요약:** 널리 쓰이는 npm, PyPI 패키지에 악성 코드가 삽입되어 개발 환경을 공격하는 사례 보고 [근거: 실시간 데이터-Hacker News/TechCrunch].
- **주요 포인트:**
  - **기술적 대응:** SBOM(Software Bill of Materials) 관리 및 종속성 검사 도구(Snyk, Dependabot) 도입.
  - **제안서 활용:** "오픈소스 라이브러리 취약점 분석 및 안전한 버전 관리 프로세스" 제공.

## 5. API 인증 우회 취약점을 이용한 무단 데이터 접근
- **핵심 요약:** REST API의 인증 로직 설계 오류(BOLA, Broken Object Level Authorization)를 통한 사용자 정보 탈취 증가 [근거: 실시간 데이터-연합뉴스 IT/The Verge].
- **주요 포인트:**
  - **기술적 대응:** 모든 API 요청에 대해 객체 수준의 접근 권한 검증 로직 구현.
  - **제안서 활용:** "OWASP API Security Top 10 기준 정밀 진단" 역량 어필.

## 6. 랜섬웨어 2.0: 데이터 암호화를 넘어선 협박(Double Extortion)
- **핵심 요약:** 데이터를 암호화할 뿐만 아니라 유출된 데이터를 공개하겠다고 협박하는 이중 협박 전략의 고도화 [근거: 실시간 데이터-ZDNet Korea/AI타임스].
- **주요 포인트:**
  - **기술적 대응:** 오프라인 백업 체계 구축 및 데이터 암호화 저장(Encryption at Rest).
  - **제안서 활용:** "침해 사고 대응(IR) 및 데이터 복구 전략 수립 컨설팅" 포함.

## 7. 스마트 컨트랙트 로직 결함을 이용한 가상자산 탈취
- **핵심 요약:** DeFi 서비스의 스마트 컨트랙트 코드 내 논리적 오류(Reentrancy 등)를 이용한 공격 발생 [근거: 실시간 데이터-TechCrunch/Hacker News].
- **주요 포인트:**
  - **기술적 대응:** 정적 분석 도구 활용 및 전문 보안 펌의 스마트 컨트랙트 오딧(Audit) 수행.
  - **제안서 활용:** "웹3.0 서비스 대상 코드 레벨 보안 취약점 점검" 확장 가능성 제시.

## 8. 사회 공학적 기법을 결합한 정교한 피싱 공격(Spear Phishing)
- **핵심 요약:** AI를 활용해 타겟의 말투를 흉내 내어 신뢰를 얻은 후 악성 파일을 설치하게 하는 정밀 공격 증가 [근거: 실시간 데이터-연합뉴스 IT/Ars Technica].
- **주요 포인트:**
  - **기술적 대응:** 다요소 인증(MFA) 의무화 및 임직원 보안 인식 교육.
  - **제안서 활용:** "기술적 진단을 넘어선 실제 공격 시나리오 기반 모의해킹" 서비스 강조.

## 9. IoT 기기 기본 패스워드 및 취약한 펌웨어 업데이트 경로 노출
- **핵심 요약:** 관리자 페이지의 기본 계정 미변경 및 암호화되지 않은 펌웨어 업데이트 경로를 통한 봇넷 형성 [근거: 실시간 데이터-넥스트데일리/The Verge].
- **주요 포인트:**
  - **기술적 대응:** 초기 패스워드 강제 변경 로직 구현 및 서명된 펌웨어 업데이트 적용.
  - **제안서 활용:** "임베디드/IoT 환경의 펌웨어 보안 분석" 역량 제시.

## 10. 제로 트러스트(Zero Trust) 아키텍처 도입 가속화
- **핵심 요약:** "아무것도 신뢰하지 않는다"는 원칙하에 모든 접근 요청을 지속적으로 검증하는 보안 모델의 기업 도입 확대 [근거: 실시간 데이터-ZDNet Korea/AI타임스].
- **주요 포인트:**
  - **기술적 대응:** 마이크로 세그멘테이션 적용 및 Identity-aware Proxy 도입.
  - **제안서 활용:** "단순 점검을 넘어 제로 트러스트 기반의 보안 아키텍처 설계" 컨설팅 제안.
